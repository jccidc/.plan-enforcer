# Public Examples

These are the fastest concrete paths through Plan Enforcer.

## 1. Full authored path

Use the built-in front door when the ask still needs to become a plan:

- [Authored path](authored-path.md)
- [Try it](../try-it.md)
- [CLI guide](../cli.md)

Short form:

```text
discuss -> draft -> review -> execute -> verify
```

## 2. Bring your own plan

Use this when planning happened elsewhere and Plan Enforcer should own
execution, verification, and closure:

- [Bring your own plan](bring-your-own-plan.md)
- `plan-enforcer import docs/plans/my-plan.md`

## 3. Composable path

Use this when you want to keep upstream planning and put the same truth
layer underneath it:

- [Composability](composability.md)
- [Composability proof](../proof/composability-proof.md)

## 4. Close and inspect

No matter how work started, the closeout band is the same:

- [Verify, audit, report](verify-audit-report.md)
- [Final truth](../proof/final-truth.md)
- [Closure surface](../proof/closure.md)

## 5. Resume continuity

Resume is part of the product loop, not cleanup after a crash:

- [Resume continuity](resume-continuity.md)
